🔐 Security & Network Architecture

This page exists for one reason: no mystery traffic. If the app touches a port, it’s listed here. If it goes outbound, it’s listed here. If you don’t use a feature, it shouldn’t even wake up.

Important: Docs are shipped inside the app. There is no “hosted documentation site”. If you’re reading this page, you’re already local. That’s the point.

🧭 Configuration Source of Truth

Networking settings are local, user-owned, and editable. No “cloud dashboard”, no remote config. 

saved/config/routing/
  └── osc_config.json        (and related network json for ports / binds)

If you change ports, change them there. If you don’t, defaults apply. Either way, services bind to 127.0.0.1 unless explicitly stated.

📘 Docs Server (In-App)

This isn’t a website. This isn’t “hosted docs”. This isn’t a SaaS pipeline. This is a local documentation viewer.

  • Served over 127.0.0.1 only
  • Used by the app’s internal docs UI
  • No external exposure
  • No telemetry

If you see it in a firewall prompt, it’s still local. Windows just loves screaming.

🦺 OwO Vest - Native SDK Transmits (Local Only)

Let’s kill the confusion:

  • The OwO Vest control path is not internet traffic
  • It’s hardware-level communication via your OS Bluetooth stack
  • That means: BLE packets, not TCP sockets

What it actually does

  • Pairs to the vest through Windows Bluetooth
  • Sends commands through BLE characteristics
  • Uses deterministic dispatch (busy locks, rate limiting, etc.)
  • Never opens internet-facing listeners
If you’re looking for an “OwO cloud endpoint” - congrats, you made one up. This stays on your box.

Visualizer Path (Optional)

If you enable a visualizer layer, that’s still local-only and configurable. It may use UDP or HTTP depending on the visualizer tool. Either way: bind stays on 127.0.0.1.

🖼 TikTok Gift Images (Outbound Pull + Cache)

Gift icons don’t appear by magic.

If a gift is unknown or not cached yet, the app can fetch the image from TikTok’s CDN over HTTPS. That is:

  • Outbound only
  • Port 443
  • Standard HTTPS GET
  • Cached locally after retrieval (so it doesn’t hammer the internet forever)

If you want zero pulls, you can ship your own asset pack and force local mapping. But if you let it fetch, it fetches - securely - like every other sane app on earth.

🔐 Internal API (127.0.0.1:8832)

This is the engine spine. The app talks to itself. Locally.

  • Chain execution
  • UI actions
  • Internal orchestration

It is:

  • Not exposed publicly
  • Not remotely routable
  • Not “a web server for strangers”

🔌 External API (127.0.0.1:8840)

This is for developer tooling, plugins, and controlled triggers - on the same machine.

  • Local-only binding
  • Strict payload validation
  • No remote access by design

If you want remote access, that’s on you. Nothing ships exposed.

⚡ PiShock Networking (Outbound Only)

PiShock requires internet because PiShock is a brokered platform. That doesn’t mean the app is “open”.

  • Outbound WSS to the broker for real-time commands
  • Outbound HTTPS for account/device discovery
  • No inbound ports opened on your machine

🧨 Safety & Failure Behavior (The Part Normies Ignore)

  • Global execution locks
  • Busy-state enforcement
  • Rate limiting + cooldowns
  • Hard stop paths
  • Reconnect with backoff (not infinite spam loops)
  • Fail-closed behavior (if it can’t validate state, it stops)

The app is built to behave like an engine, not a toy script.

✅ Final Word

If you’re paranoid: good. This doc is for you.

Stream Connector is:

  • Local-first
  • Explicit-by-design
  • No inbound internet exposure
  • Auditable network surface

If it’s not listed here, it shouldn’t exist. And if it does exist, it’s your cue to yell at the developer (hi) and make it listable.

No mystery sockets. No hidden listeners. No bullshit.
