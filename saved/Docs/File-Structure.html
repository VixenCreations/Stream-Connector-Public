File Structure

This is how Stream Connector is laid out. Nothing here is accidental. Everything exists for a reason.

Root

Stream Connector/
├── Stream Connector.exe
├── giftMapping.json
└── saved/

Delete saved/ and the app resets. Everything important lives there.

saved/

saved/
├── changelogs/
├── config/
├── controls/
├── logs/
└── userdata/

saved/changelogs/

saved/changelogs/
└── version.json

Used by the UI to display release history.

saved/config/

saved/config/
├── dev.json            (debug toggle: picks logs_debug.db vs logs_prod.db)
├── user.json
├── devices/
│   ├── devices.json
│   └── intiface_devices.json
├── filters/
│   ├── noisy_parameters.json
│   └── nuclear.json
├── owo/
└── routing/
    └── endpoints.json

saved/controls/

saved/controls/
├── chains/
│   ├── *.json
│   └── backup/
├── export/
├── owo/
└── backup/

saved/logs/ (SQLite)

Logging is now a pair of SQLite databases (one table per module), not loose .log text files. dev.json decides which one is used. 

saved/logs/
├── logs_debug.db   ← debug runs (dev.json debug_mode = true; appended)
└── logs_prod.db    ← normal runs (truncated each launch)

saved/userdata/

saved/userdata/
└── license.json
  • Generated automatically
  • Required for startup
  • Regenerated if missing

Final Notes

  • Everything persistent lives in saved/
  • Logs are cleanly separated by subsystem
  • No silent failures
  • If something breaks, the logs will tell you why
