🌐 Overview

Stream Connector includes a first-class Streamer.bot integration layer designed for low-latency, strongly-typed event handling with full safety controls.

This system allows Streamer.bot actions to trigger internal engine logic, device execution, and automation workflows without exposing unsafe endpoints or requiring plugins.

🔌 Connection Details

Property Value
Protocol WebSocket
Address ws://127.0.0.1:8080
Direction Local only
Source Streamer.bot
Mode Subscribe → Receive → Execute
Auth None (localhost only)

📡 Subscription Model

Stream Connector subscribes to Streamer.bot custom events:

{
  "request": "Subscribe",
  "id": "streamconnector-sub",
  "events": {
    "General": ["Custom"]
  }
}

Only General → Custom events are processed.

🧱 Event Model

All events follow a normalized structure:

{
  "provider": "string",
  "commandId": "string",
  "context": {}
}

These are parsed into strongly-typed objects before execution.

🧩 Supported Event Types

External Hook Event

Used for generic external triggers.

Fields: - provider - command_id - timestamp - user - message - source

Twitch Chat Event

Triggered on chat input.

Fields: - user - message

Twitch Subscription Event

Fields: - user - tier - months - is_gift - gifter

Twitch Cheer Event

Fields: - user - bits - message

OBS Scene Event

Fields: - scene

Timer Event

Fields: - timer

Button Event

Fields: - button

🔁 Execution Flow

Streamer.bot
→ WebSocket
→ Event Validation
→ Typed Parser
→ Execution Router
→ Stream Connector Core

⚙️ Execution Hook

All parsed events are routed through:

execute_external_hook(provider, command_id, context)

This allows: - Device control - Pattern execution - OSC output - External integrations
