🌐 External Integration API

This is the **local-only control surface** for Stream Connector. No cloud. No bullshit. No magic.

If something triggers an action from the outside world - it comes through here.

🔌 Server Overview

ProtocolHTTP
Host127.0.0.1
Port8840
ScopeLocalhost only
AuthNone (local trust boundary)
CORSEnabled (local tools)

Nothing here is internet-facing. Nothing opens inbound ports. Nothing phones home.

📡 Available Endpoints

GET /api/external/info

Returns basic metadata about the API.

{
  "data": {
    "name": "Stream Connector External API",
    "author": "Vixenlicious",
    "version": "x.y.z"
  }
}

GET /api/external/list

Returns all registered external hooks.

{
  "data": [
    {
      "provider": "example",
      "commandId": "trigger_thing",
      "description": "Example external hook"
    }
  ]
}

POST /api/external/exec

Executes a hook. Supports JSON, form-data, and query strings.

🧩 Request Format

JSON (recommended)

{
  "provider": "example_provider",
  "commandId": "example_command",
  "context": {
    "user": "viewer123",
    "message": "hello world"
  }
}

Form / Query Fallback

provider=example
commandId=test

🧠 Context Handling

  • Passed through untouched
  • No evaluation
  • No execution
  • No serialization hacks
  • Safe structured data only

It’s payload, not code.

⚙️ Execution Flow


External Tool
   ↓
HTTP Request (localhost)
   ↓
Validation
   ↓
Execution Router
   ↓
Stream Connector Core

🔐 Security Model

  • ✔ Localhost only
  • ✔ No inbound exposure
  • ✔ No eval / exec
  • ✔ No telemetry
  • ✔ Graceful failure
  • ✔ Input normalization

🚨 Error Handling

Errors always return HTTP 200 to avoid breaking bots and automations. 

{
  "ok": false,
  "error": "provider and commandId required"
}

🧪 Example (Python)


import requests

requests.post(
  "http://127.0.0.1:8840/api/external/exec",
  json={
    "provider": "demo",
    "commandId": "fire_effect",
    "context": {"user": "viewer123"}
  }
)

🛡️ Guarantees

  • No remote control
  • No cloud traffic
  • No code execution
  • No telemetry
  • No silent failures

✅ Summary

This API exists for one reason:

Let external tools trigger controlled actions without breaking your system.

It is local. It is predictable. It does not do dumb shit.
